# Implementation Plan — Video-to-Script MVP

> Last updated: 2026-04-12

---

## ✅ Completed

### Auth & Session Management
- [x] **Zombie Cookie Fix** — Replaced client-side `supabase.auth.signOut()` with a Next.js Server Action (`logout()` in `app/login/actions.ts`) that forcibly destroys the session cookie and revalidates the cache. Logout button wrapped in `<form action={logout}>`.
- [x] **Anonymous / Guest Auth** — `useGuestAuth` hook auto-signs in visitors anonymously via `supabase.auth.signInAnonymously()`. Credit balance read from `profiles` table. Anonymous Auth toggled ON in Supabase Dashboard.
- [x] **Middleware Route Guard** — `utils/supabase/middleware.ts` blocks all routes except `/`, `/login`, and `/auth` for unauthenticated users. Root path (`/`) explicitly allows unauthenticated access so `useGuestAuth` can trigger.
- [x] **Orphan Account Guard** — Middleware detects non-anonymous users with no `profiles` row and signs them out automatically.

### Credit System
- [x] **Credit Badge** — `CreditBadge` component shows `✦ N analyses left` for guest users. Hidden for registered users.
- [x] **Credit Decrement** — API routes (`/api/generate-script`, `/api/generate-hashtags`) decrement credits *after* successful generation to avoid double-charging on error.
- [x] **Credit Exhaustion** — 402 response triggers `UnlockCTA` / upgrade toast with "Unlock 50 Credits →" CTA. `UsageGuard` component renders blocking card at 0 credits.
- [x] **Credit Refresh** — `refreshCredits()` called after successful generation to update the badge without page reload.

### Video Upload & Storage
- [x] **Signed Upload URL** — `/api/upload-url` generates a Supabase signed upload URL. Client PUTs directly to storage using native `fetch` (not the Supabase SDK, which was silently hanging).
- [x] **Video Optimisation** — `needsOptimization()` + `optimizeVideoForAI()` compress large/4K files via FFmpeg.wasm before upload.
- [x] **Upload Deduplication** — `uploadVideoIfNeeded()` checks `uploadedVideoUrl` state before re-uploading. Shared via `StudioContext` across pages.
- [x] **File Size Guard** — `handleFileSelect` in `app/hashtags/page.tsx` rejects files over 100MB with a toast error.
- [x] **MIME Type Fallback** — Upload headers use `fileToUpload.type || 'video/mp4'` to handle missing MIME types robustly.

### Script Generation
- [x] **Multi-slot Generation** — Aesthetic, Funny, Educational, and Custom AI script slots all share the same `handleGenerateScript` flow.
- [x] **Custom AI Builder** — Custom prompt textarea + "Generate Custom Script" button in `ScriptSidebar`. Shares `uploadVideoIfNeeded` with standard slots.
- [x] **Script Overlay Editor** — Segment editing with `pendingEdits`, save/undo per segment, "Save All" batch commit.
- [x] **Script Persistence** — Generated scripts persisted to `localStorage` via `StudioContext`. Rehydrated on mount.

### Hashtag Studio
- [x] **`/hashtags` Route** — Full Hashtag Studio page with upload zone, AI analysis via `/api/generate-hashtags`, chip-based keep/remove UI.
- [x] **Cultural Vibes Tab** — `cultural` hashtag group generated by Gemini and displayed in its own panel.
- [x] **Kept Hashtags Persistence** — Only `kept` array persisted to `localStorage` (transient suggestions are not).
- [x] **User Hint Input** — Optional context textarea to personalise hashtag generation (e.g. "Dubai rooftop golden hour").
- [x] **Get More Suggestions** — "Get More Suggestions" button passes `excludeHashtags` to avoid duplication.
- [x] **Copy All** — Clipboard copy of all kept hashtags.
- [x] **Cross-page Video Context** — `StudioContext` shares `uploadedVideoUrl` between Script Studio and Hashtag Studio. Banner shown if video carried over.

### Security
- [x] **Auth Guards on all API routes** — All `/api/*` routes verify the Supabase session cookie before processing.
- [x] **SSRF Prevention (partial)** — `fileUrl` from client is used to fetch video server-side. The API validates the URL hostname against `NEXT_PUBLIC_SUPABASE_URL` before fetching. (See Open Items.)

### DevOps & Testing
- [x] **Playwright E2E** — `e2e/auth.spec.ts` verifies login → logout → session destruction.
- [x] **Guest Tier Test** — `e2e/guest-tier.spec.ts` validates anonymous auth, credit badge display, file upload, and custom AI flow.
- [x] **QA Tester Workflow** — `.agent/workflows/qa-tester.md` documents the automated testing process for new features.
- [x] **`npm run build` passing** — TypeScript type error in `HashtagPayload` cast (`cultural` field missing) fixed in `app/api/generate-hashtags/route.ts`.
